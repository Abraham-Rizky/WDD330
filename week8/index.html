Notes


  1. Translation in CSS allows us to move elements left, right, up, or down
  2. The scale function scales an element by defined factors horizontally then vertically. For example: scale(1.5, 0.25)
  3. The rotate function rotates an element around the point of origin by a specified angle value
  4. Rotate and scale function has the default point of origin of the center of element. For example: transform: rotate (10deg)
  5. The skew(x,y) function specifies a skew along the x and y axes
  6. CSS transitions generally require less client-side processing, so they’ll usually appear smoother
  7. Transition-property is really useful to define which element should be transitioned. By default all properties are transitioned
  8. 0.2s or 200ms is generally considered the optimum time for a transition
  9. The transition properties allow for multiple transitions in one call. For example, if we want to change the color at the same time as changing the rotation and size, we can
  10. With HTML5’s Canvas API, we can draw anything we can imagine, all through JavaScript
  11. Since drawing on the canvas is done using JavaScript, we’ll need a way to grab the element from the DOM
  12. In the canvas coordinate system, the top-left corner is (0,0). If the canvas is 200 pixels by 200 pixels, then the bottom-right corner is (200,200)
  13. Unlike canvas, images created with SVG are available via the DOM
  14. Canvas allows for pixel manipulation compared to SVG
