Notes


  1. Client side validation is like an initial check to catch invalid data on the client side which help the users so they may fix it right away
  2. A couple ways to do client side validation are built-in form validation using HTML 5 and JavaScript
  3. A regular expression (regex) is a pattern that can be used to match character combinations in text strings, so regexps are ideal for form validation and serve a variety of other uses in JavaScript
  4. fetch() won’t reject on HTTP error status even if the response is an HTTP 404 or 500. Instead, it will resolve normally (with ok status set to false), and it will only reject on network failure or if anything prevented the request from completing
  5. fetch() won’t send cross-origin cookies unless you set the credentials init option
  6. An accurate check for a successful fetch() would include checking that the promise resolved, then checking that the Response.ok property has a value of true
  7. Instead of passing a path to the resource we want to request into the fetch() call, we can create a request object using the Request() constructor, and pass that in as a fetch() method argument
